# StatikAnalyzer 要件定義書

## 1. システム名

StatikAnalyzer

## 2. システム概要

StatikAnalyzerは、ファイルを実行せずに静的解析を行い、ファイルの基本情報、ハッシュ値、PEヘッダー情報、セクション情報、VirusTotal APIによる外部評価を統合することで、不審ファイルの一時的な切り分けを支援するシステムである。

本システムでは、解析結果をJSON形式で保存し、時系列タイムライン上で確認できるようにする。これにより、どの時刻にどのファイルが解析され、どのような根拠で不審と判断されたのかを追跡可能にする。

## 3. 開発目的

本システムの目的は、インシデント調査やフォレンジック調査の初期段階において、調査対象となるファイルの優先順位付けを支援することである。

従来、ファイル単体の解析結果は個別に確認されることが多いが、本システムでは解析結果を時系列イベントとして扱うことで、ログ解析やインシデントタイムラインとの統合を見据えた構成とする。

## 4. 対象ファイル

本システムでは、主にWindows PEファイルを解析対象とする。

対象とするファイルは以下の通りである。

* `.exe`
* `.dll`
* `.sys`
* その他、PE形式の可能性があるファイル

また、非PEファイルについても、PEではないことを判定し、解析結果として記録する。

## 5. 機能要件

### 5.1 ファイル基本情報取得機能

指定されたファイルについて、以下の情報を取得する。

* ファイル名
* ファイルパス
* ファイルサイズ
* 拡張子
* 作成時刻
* 更新時刻

### 5.2 ハッシュ計算機能

指定されたファイルについて、以下のハッシュ値を計算する。

* MD5
* SHA-256

SHA-256はVirusTotal API照会時の識別子として使用する。

### 5.3 PEファイル判定機能

対象ファイルがPE形式であるかを判定する。

判定には以下を用いる。

* DOS Headerの`MZ`シグネチャ
* `e_lfanew`によるPE Header位置
* PE Signatureの`PE\0\0`

### 5.4 PEヘッダー解析機能

PEファイルであると判定された場合、以下の情報を取得する。

* Machine
* NumberOfSections
* TimeDateStamp
* SizeOfOptionalHeader
* Characteristics
* AddressOfEntryPoint
* ImageBase
* Subsystem

### 5.5 セクション情報解析機能

PEファイルのSection Tableから、以下の情報を取得する。

* セクション名
* VirtualSize
* SizeOfRawData
* PointerToRawData
* Characteristics
* Entropy

### 5.6 VirusTotal API連携機能

SHA-256ハッシュを用いてVirusTotal APIに照会し、既知ファイルであるかを確認する。

取得する情報は以下の通りである。

* 照会成功可否
* malicious数
* suspicious数
* harmless数
* undetected数
* 最終解析日時

MVPでは、ファイル本体のアップロードは行わず、ハッシュ照会を基本とする。

### 5.7 分類機能

解析結果をもとに、対象ファイルを以下のいずれかに分類する。

* clean
* unknown
* suspicious
* malicious_candidate
* non_pe
* error

分類は、VirusTotalの検知数、PE構造の異常、セクションエントロピーなどをもとにルールベースで行う。

### 5.8 JSON出力機能

解析結果をJSON形式で保存する。

JSONには、ファイル基本情報、PE解析結果、VirusTotal照会結果、分類結果を含める。

### 5.9 タイムライン出力機能

解析イベントを時系列タイムライン用JSONとして出力する。

各イベントには以下を含める。

* timestamp
* source
* event_type
* file_name
* sha256
* classification
* risk_score
* message

## 6. 非機能要件

### 6.1 安全性

本システムでは、対象ファイルを実行しない。解析は静的解析に限定する。

### 6.2 プライバシー

VirusTotal API連携では、原則としてファイル本体をアップロードせず、ハッシュ値のみを照会する。

### 6.3 APIキー管理

VirusTotal APIキーは`.env`ファイルで管理し、GitHubなどの外部リポジトリには公開しない。

### 6.4 拡張性

将来的にATTやAI-SOC基盤と連携できるよう、解析結果はJSON形式で保存する。

### 6.5 例外耐性

壊れたPEファイル、非PEファイル、読み取り不能ファイル、APIエラーが発生しても、システム全体が停止しないようにする。

## 7. MVPの完成条件

本システムのMVP完成条件は以下とする。

* 単一ファイルを指定して解析できる
* ファイルのハッシュ値を計算できる
* PEファイルかどうか判定できる
* PEヘッダー情報を取得できる
* セクション情報を取得できる
* VirusTotal APIでSHA-256ハッシュ照会ができる
* 解析結果をJSONで保存できる
* タイムライン表示用JSONを生成できる
* 正常ファイル、非PEファイル、壊れたファイルに対して処理が停止しない

## 8. 想定する分類結果

| 分類                  | 内容                     |
| ------------------- | ---------------------- |
| clean               | 明確な不審点が少ないファイル         |
| unknown             | 情報不足で判断できないファイル        |
| suspicious          | 構造や特徴量に不自然な点があるファイル    |
| malicious_candidate | VirusTotalで複数検知があるファイル |
| non_pe              | PE形式ではないファイル           |
| error               | 解析中にエラーが発生したファイル       |
