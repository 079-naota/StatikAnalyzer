# StatikAnalyzer 基本設計書

## 1. システム全体の処理フロー

StatikAnalyzerの処理フローを以下に示す。

```text
ファイルの読み込み
ファイル情報の抽出
ファイル基本情報の整理
ハッシュ値の計算
PEファイルかどうかの判定
PEならヘッダー情報を取得
PEならセクション情報を取得
セクションごとのエントロピーを計算
VirusTotalに問い合わせる
分類を行う
Jsonスキーマに合わせ保存する
タイムラインように変換する
```

## 2. モジュール構成

本システムは、以下のモジュールで構成する。

| モジュール名            | 役割 |
| ----------------- | -- |
| main              | 全モジュールの呼び出し |
| file_loader       | ファイルの読み込み     |
| hash_calc   | ハッシュ計算   |
| pe_parser         | PE情報の抽出と整理   |
| entropy           |    |
| virus_client | VirusTotal APIでの照会   |
| classifier        | 抽出した情報の分類   |
| event_builder     | イベントのタイムライン化   |
| exporter          | Jsonスキーマに沿った出力の結果   |

## 3. 各モジュールの設計

### 3.1 main

役割:　各モジュールの呼び出し

入力:　実行命令とファイル

出力:　Jsonファイル

失敗時の扱い:　プログラム全体を狩猟せずに、解析結果Jsonにエラー内容を記録する

### 3.2 file_loader

役割:　渡されたファイルの読み込みと基本情報の抽出

入力:　ファイル

出力:　ファイルを読み込めたかどうか、ファイルの基本情報

失敗時の扱い:　ファイルがない場合は「None」、読み込めない場合は「read_error」、権限がない場合は「permission_denied」 

### 3.3 hash_calc

役割:　基本情報から得たハッシュ値の計算

入力:　ファイル

出力:　ハッシュ値（SHA-256 and MD5）

失敗時の扱い:　計算不可の場合は「Not Found」で０を返す

### 3.4 pe_parser

役割:　PEファイルかどうかの判定とPE情報の抽出

入力:　ファイル

出力:　PEかどうか、ヘッダー情報、セクション情報

失敗時の扱い:　PE判定が０の場合はPEではないという情報を返す、PEが壊れている場合は注意メッセージを追加する、１の場合は情報を辞書で返す

### 3.5 entropy

役割:　PEセクションからエントロピーを計算し、セクションごとの不自然さをみる数値を出す

入力:　セクション名、セクション生データ、データサイズ

出力:　エントロピー値、エントロピーの高低

失敗時の扱い:　セクションデータが読み込めない場合「None」でエラー内容を書く

### 3.6 virus_client

役割:　VirusTotal　APIへのハッシュ値の問い合わせ

入力:　計算後のハッシュ値

出力:　結果を辞書で返す

失敗時の扱い: 通信エラーの場合は「impossible」、問い合わせ結果がない場合は「Not Found」、　ある場合は。内容を辞書として返す

### 3.7 classifier

役割:　ファイルの基本情報、エントロピー、VT結果をもとに、ファイルの分類とLevelを決める

入力:　各モジュールで得た情報

出力:　ファイルの分類、Level、判定理由

失敗時の扱い:　分類不可の場合は「unknown_error」

### 3.8 event_builder

役割:　静的解析段階のタイムライン生成

入力:　解析結果Json、level、解析時刻、ファイル名

出力:　タイムライン用のイベント辞書

失敗時の扱い:　イベント生成のどの段階でエラーが出たかを示す

### 3.9 exporter

役割:　Jsonスキーマに沿った結果を指定パスに保存する

入力:　保存したい情報（ファイル名、データ）

出力:　保存の有無、保存先のパス

失敗時の扱い:　保存先ディレクトリがない場合「Not　Found」、権限なし「Permission_denied」、Jsonに変化できないデータの場合は「Impossible」

## 4. 解析結果JSONの設計

解析結果JSONには、以下の情報を含める。

```json
{
  "file_info": {},
  "hashes": {},
  "pe_info": {},
  "vt_result": {},
  "classification": {},
  "errors": []
}
```

## 5. タイムラインJSONの設計

タイムライン表示用JSONには、以下の情報を含める。

```json
{
  "timestamp": "",
  "source": "StatikAnalyzer",
  "event_type": "",
  "file_name": "",
  "sha256": "",
  "classification": "",
  "level": 0,
  "message": ""
}
```

## 6. 例外処理方針

本システムでは、解析中にエラーが発生しても、可能な限り処理を継続する。

想定するエラーは以下の通りである。

* ファイルが存在しない
* ファイルを読み取れない
* PE形式ではない
* PE構造が壊れている
* VirusTotal APIキーが存在しない
* VirusTotal APIの制限に達した
* ネットワークエラー

エラーが発生した場合は、プログラム全体を停止せず、解析結果JSONの`errors`に記録する。

## 7. MVPで優先する設計方針

本システムでは、MVP段階において以下を優先する。

* 単一ファイル解析を優先する
* ファイル本体は実行しない
* VirusTotalにはハッシュのみ照会する
* 解析結果は必ずJSONとして保存する
* エラーも解析結果の一部として扱う
* 後からATTやAI-SOC基盤と接続しやすい構造にする
